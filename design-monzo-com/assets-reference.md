## Phone Bezel Assets

Use the Phone Bezel component to showcase app UI within a device frame. Background colours are controlled via variants on the component — do not create custom backgrounds.

| Component Name | Component Key |

## Other Asset Types

The following asset types require manual design work from the user. When planning, suggest these as text directions only — do not attempt to assemble them in Figma.

### Simplified UI on Solid Background
Extracted UI elements (e.g. a card, notification, or feature snippet) placed on a flat brand-colour background. Strip away nav bars, status bars, and controls — focus on the single key element. Backgrounds must be **Hot Coral** or **Dark Navy** only. No textures, patterns, gradients (except DS "Subs" gradients for subscription products), or light grey/off-white.

### Lifestyle Photography + UI Overlay
A photography component used as the background with UI elements, text overlays, or illustrative graphics layered on top. Use DS elevation (shadow) styles only — no heavy black shadows. Frame the composition so the UI and photo complement each other. **Always suggest and insert the photography component** from the table above — the user only needs to manually add the UI overlay on top.

### Lifestyle Photography + Physical Card
A photography component as background with a physical Monzo card composited on top. Frame so the card doesn't cover the photo's visual focus. **Always suggest and insert the photography component** from the table above — the user only needs to manually composite the card.

### Colour Rules for Assets in Rows
When multiple assets appear in a row (card carousels, feature highlights, linked articles), never use Hot Coral as a standalone background. Alternate using these patterns:
- Hot Coral / Photography / Hot Coral
- Hot Coral / Dark Navy / Hot Coral
- Photography / Hot Coral / Photography / Hot Coral
